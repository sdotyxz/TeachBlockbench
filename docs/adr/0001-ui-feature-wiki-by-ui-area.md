# Organize the UI feature wiki by visible UI area

The generated Blockbench UI feature reference is split into a wiki-like directory of static HTML
pages organized by the top-level UI area that starts each UI path, such as File, View, UV, Timeline,
or Mode Switcher. Features that are reachable from multiple UI areas appear on every matching page
because learners browse from where they are in Blockbench, not from extractor feature types or source
modules.
