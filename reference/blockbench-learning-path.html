Lesson Trail

Lessons 1–4 — Modeling foundation

You built a crate, chopping board with knife, cooking pot, and plate with spoon. The lasting skill is not the props themselves; it is the loop: add cubes, resize, move, duplicate, rotate, name parts, frame a render, and post.

Lessons 5–7 — First texturing foundation

You moved from random colors to a shared palette, then from flat color to painted pixels, then from painting to UV placement. This is the start of making assets look like one pack instead of separate exercises.

Lesson 8 — Shape polish

Inflate / Deflate belongs mostly to modeling, but it protects texture work too because it can fix tiny face overlap without redrawing UVs. This is the right next lesson after your first UV mapping win.

Lessons 9–16 — Reliable prop production

Build more kitchen props while adding one new skill each time: mirror modeling, grouping, better UV template use, texture layers, labels, screenshots, and export sanity checks.

Lessons 17–24 — Character and animation

Make a simple chef, learn pivot-aware posing, create one very small animation loop, and render a moving postable artifact. Keep animation simple until the static pack is strong.

Lessons 25–36 — Modular pack and ship

Build counters, floors, wall pieces, variants, export files, preview images, license text, and the itch.io product page. This is where Blockbench skill turns into a product.

Settings Tutorial Pattern

Each lesson should include a short settings section with this shape. Keep it practical; settings are only taught when they unlock the day's artifact.

  1. Setting name and location

    Example: Fill Mode in the Paint tool settings.

  2. Beginner value

    Example: use Element or Cube when recoloring a whole part.

  3. What changes on screen

    Example: the brush affects one face, one connected color area, or the whole selected element.

  4. When to change it

    Example: switch to Face for one side of a sign; switch back to Element for fast palette cleanup.

Settings Index

This is the settings spine for the course. Completed lessons can be updated when revisited; new lessons should include the settings focus from the start.

07 — First UV mapping UV panel face tabs, UV rectangle movement/resizing, UV rotation only if needed. Map one important face first; do not unwrap the whole prop in one lesson.
08 — Inflate / Deflate Element panel Inflate value and tiny negative values for overlap fixes. Use small values; check from multiple angles before increasing them.
09 — Label/decals UV grid/snap, texture resolution, face-specific UV placement. Use one label patch and one selected face; keep text large enough to read.
10 — Modular tile Grid size, snap increments, typed transforms, duplicate spacing. Keep module dimensions round and repeatable.

Source Code Anchors

The Blockbench repo is cloned locally at reference/blockbench-source. These links use the pulled commit 8fe8d9d so the learning map can be checked against real source code. The GitHub links work even when this page is published without the local clone.

ConceptSource anchorWhy it matters for lessons
Modes js/modes.ts Edit, Paint, and Animate are not just labels; Blockbench switches visible tools, panels, and selection behavior by mode.
Cubes and faces js/outliner/types/cube.js A cube stores shape, rotation, origin, inflate, and six faces. This explains why modeling and UVs meet at the cube face.
Move and rotate js/modeling/transform.js The early lessons correctly focus on transforms first because they are the core operations behind prop construction.
Textures js/texturing/textures.js A texture is an image/canvas plus material data. This supports the lesson distinction between color, image pixels, and UV placement.
Painting js/texturing/painter.js Painting edits bitmap texture data. That is why Lesson 6 changed the crate's pixels without changing its geometry.
UV editor js/uv/uv.js The UV editor connects texture coordinates to faces. This validates Lesson 7's "move the face UV rectangle" mental model.
Animations js/animations/animation.js An animation owns length, loop behavior, snapping, and animators. This is the map for the first animation lessons.
Keyframes js/animations/keyframe.js A keyframe stores time, channel, and values. The first animation lesson should teach only this much before interpolation.
Timeline js/animations/timeline.js The timeline tracks time, selected keyframes, playback speed, and animation length. Settings tutorials should cover these gradually.
Tool labels and settings lang/en.json The language file exposes beginner-facing settings names like Inflate, Fill Mode, Brush Size, Opacity, and Softness.
How to read this page before each lesson

Find the current lesson's pillar, check the next missing skill, then make sure the lesson has one small settings tutorial. If a proposed lesson does not advance one of the three pillars, save it for practice instead of making it the main daily lesson.
