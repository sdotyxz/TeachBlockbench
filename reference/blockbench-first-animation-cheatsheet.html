Animation reference

First Animation Cheatsheet

A beginner-safe map for the first Blockbench Timeline loop: one moving group, three position keyframes, looped playback, and a GIF for posting.

Mental model

A keyframe is a saved pose at a specific time. The Timeline plays between keyframes, so a three-keyframe bounce can be: down at 0.0s, up at 0.5s, down again at 1.0s.

Core controls

ControlBeginner meaningUse today
Animate modeThe workspace for editing animation instead of the base model.Switch here after the model is built.
Animations panelWhere animations are added, named, selected, and given a length.Add one animation named tomato_bounce.
Timeline panelThe time ruler where keyframes appear.Place keyframes at 0.0, 0.5, and 1.0 seconds.
Position channelThe track that stores where the selected group is located.Key only Position for the first lesson.
Looped PlaybackPreviews the animation repeatedly.Turn it on to check that the start and end match.

Beginner rules

  • Animate one grouped prop first. Do not animate every cube separately.
  • Make the first and last keyframe identical for a clean loop.
  • Use Position before Rotation or Scale. It is the easiest channel to debug.
  • If playback jumps at the end, copy the first pose to the final keyframe.
  • If nothing moves, check that the correct group is selected in Animate mode.

Troubleshooting

SymptomLikely causeFix
No Timeline appears.The panel is hidden or you are not in Animate mode.Switch to Animate mode, then use View > Panels > Timeline.
The tomato moves in the model forever.You edited in Edit mode instead of adding a keyframe in Animate mode.Undo, switch to Animate mode, then move at a selected time.
Only one cube moves.The tomato parts were not grouped, or the wrong element is selected.Select the moving group, not an individual cube.
The loop pops at the end.The last keyframe differs from the first.Set the final pose to match the first pose exactly.
