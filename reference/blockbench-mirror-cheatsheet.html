Blockbench Mirror Cheatsheet

A quick map for using Mirror Modeling safely: what it mirrors, when to apply it, and how it differs from Duplicate, Flip, and Mirror UV.

Mental model

Mirror Modeling reflects edits across the X axis. Build on one side of the model, check the mirrored side, then use Apply Mirror Modeling when you want the reflected side to become real editable geometry.

Main controls

ControlWhereWhat it doesBeginner use
Mirror ModelingEdit > Mirror ModelingLive-reflects viewport edits across X.Turn on before building one side.
Apply Mirror ModelingEdit > Apply Mirror ModelingCopies selected elements and flips them across X.Use when the mirrored side looks right.
Allow Mirror ModelingEdit mode, selected elementLets a selected element participate in mirroring.Leave on unless one part should not mirror.
Mirror UVMirror Modeling options / UV panelMirrors texture coordinates too.Keep on for simple solid-color props.

Mirror vs nearby tools

ToolUse whenBeginner trap
DuplicateYou want a second copy you will place manually.The copy is not automatically opposite or mirrored.
Mirror ModelingYou want edits on one side reflected live across X.The center line matters. If the model is off-center, the mirror looks wrong.
Apply Mirror ModelingYou are done previewing and want real copied parts.Apply only the parts you want mirrored.
Transform > FlipYou want to flip selected geometry on X, Y, or Z.Flip is a direct transform, not the same as live Mirror Modeling.
Mirror UVYou want texture layout mirrored.This changes texture behavior, not model shape.

Safe beginner workflow

  1. Center the main prop around X 0

    Mirror Modeling reflects across the X center line, so a centered body gives predictable pairs.

  2. Turn on Edit > Mirror Modeling

    Keep Mirror UV on for simple solid-color props.

  3. Build only one side

    Watch the reflected side appear. If it appears in the wrong place, fix the original side's position relative to the center line.

  4. Use Apply Mirror Modeling

    Select the one-side pieces and apply when you want the mirrored copy to become editable.

  5. Turn Mirror Modeling off

    After applying, turn it off so later asymmetrical details do not duplicate by accident.

Troubleshooting

SymptomLikely causeFix
The mirror appears too close or too far.The original part is not placed relative to X 0 as expected.Move the original side and watch the reflected side update.
A center piece duplicates unexpectedly.Mirror Modeling is still on while editing a part that should stay single.Turn Mirror Modeling off or disable mirroring for that element.
The mirrored side is only a preview.You have not applied Mirror Modeling.Select the source side and choose Edit > Apply Mirror Modeling.
Textures look reversed.Mirror UV is affecting the mirrored faces.For text/labels, turn off Mirror UV or clean the UVs after applying.
Source-backed facts

The local Blockbench UI atlas lists Edit > Mirror Modeling as a toggle that reflects viewport changes across the X axis, and Apply Mirror Modeling as an action that copies selected elements and flips them across X.
