# itch.io Devlog Block Kit

Use this kit when a lesson ends with an itch.io post. The goal is a fun visual devlog, not a lesson
transcript. The course page can hold the full teaching detail; itch.io should show clear proof of
progress, a few punchy observations, and a reason to come back tomorrow.

itch.io's editor is rich text, not raw Markdown. Draft the post here, then paste plain text into the
editor and apply headings, lists, links, and images with the toolbar.

## Working Rule

Every post should use 4-6 blocks from this kit:

1. Lead with visible progress.
2. Keep each text block short.
3. Put images near the point they explain.
4. End with a next-step hook.

Do not use every block every day. Pick the blocks that fit the artifact.

## Core Blocks

### 1. Hero Proof

Use when: every post.

Purpose: show the day's result before explaining it.

Ingredients:

- One strong image near the top.
- One 1-2 sentence caption.
- If possible, use a before/after composite instead of a single render.

Example:

> Day 5 proof: the same four kitchen props, but now they share one palette. The shapes did not
> change. The set suddenly feels more intentional.

Image requirement:

- Wide or square render.
- Cropped close enough that the change is obvious at feed size.
- If it is a before/after, label the two sides in the image or the caption.

### 2. Tiny Win

Use when: every post.

Purpose: summarize the lesson in one satisfying sentence.

Format:

> Today's tiny win: I changed [small thing], and now [visible result].

Examples:

> Today's tiny win: I reused the exact same wood and metal colors across four props, and now they
> look like they belong in one pack.

> Today's tiny win: I painted three plank lines on the old crate, and now the flat cube reads more
> like wood.

### 3. Before / After

Use when: the lesson improves an existing asset.

Purpose: make progress instantly legible.

Format:

- Image: before/after composite.
- Caption: "Before: [problem]. After: [specific improvement]."

Example:

> Before: four useful props, but each one felt separate. After: shared wood, shared metal, shared
> cream, and a small warm accent tying them together.

### 4. Visual Breakdown

Use when: the post has 2-4 visible changes worth pointing out.

Purpose: replace paragraphs with a quick scan.

Format:

- Short heading: "What changed"
- 3-4 bullets.
- Each bullet starts with the visible part, not the abstract lesson.

Examples:

- Wood: crate, board, and handles now use the same brown.
- Metal: pot, spoon, and knife share one steel color.
- Cream: the plate color repeats as a soft highlight.
- Accent: tomato red and leaf green add warmth without making a new palette.

### 5. Palette Strip

Use when: color is part of the lesson.

Purpose: make the palette feel like a visual asset, not a code list.

Format:

- Add a small image showing the swatches if possible.
- Under it, list only name + role. Keep HEX codes optional or small.

Example:

> Six colors did the job: cream for ceramic, wood for bodies and handles, dark wood for edges, steel
> for tools, tomato for warmth, and leaf green for a tiny fresh accent.

Image requirement:

- A horizontal strip with labeled swatches is better than a bullet-only list.
- If no strip exists yet, use the course palette reference as the source and keep the post text short.

### 6. Three-Step Workflow

Use when: the process is interesting but should not become a tutorial.

Purpose: show how the result happened without recreating the whole lesson.

Format:

1. Picked [constraint/tool].
2. Changed [asset area].
3. Checked [visual result].

Example:

1. Loaded the Endesga64 course palette.
2. Refilled each prop from that same palette.
3. Put the four props side by side to check whether they read as one pack.

### 7. Mistake / Fix

Use when: there was a useful beginner trap.

Purpose: make the post more human and more useful.

Format:

> The mistake: [temptation].
> The fix: [small rule].

Examples:

> The mistake: eyeballing "close enough" browns for each model.
> The fix: type the same HEX value every time.

> The mistake: adding too many wood marks until the crate looked noisy.
> The fix: zoom out and delete marks until the plank read is still clear.

### 8. Thumbnail Test

Use when: the lesson is about readability, composition, texture, or marketplace presentation.

Purpose: connect the work to selling an asset pack.

Format:

> Thumbnail test: when I zoom out, can I still tell what changed?

Example:

> Thumbnail test: the new palette works because the props still separate clearly, but the repeated
> colors make the group read as one product image.

### 9. Next Up

Use when: every post.

Purpose: create continuity.

Format:

> Next up: [one concrete next asset or skill].

Examples:

> Next up: painting the first real pixel texture on the crate.

> Next up: building a chalkboard menu stand and learning first UV mapping.

## Optional Blocks

### Mini Gallery

Use when: there are multiple good renders.

Format:

- One final hero image.
- 2-3 smaller images below it.
- One caption per image.

Keep it visual. Do not repeat the same explanation under every image.

### Source Of Truth

Use when: the post mentions a reusable constraint like a palette, naming system, or export rule.

Format:

> Rule I am keeping from now on: [simple rule].

Example:

> Rule I am keeping from now on: the shared palette is the source of truth for every kitchen prop.

### Progress Streak

Use when: the daily streak matters.

Format:

> Streak: Day [N]. One small public artifact per day.

Use once, near the end.

## Filled Example: Day 5 Shared Palette

Title:

> Day 5: One palette makes four props feel like a pack

Blocks to use:

- Hero Proof
- Tiny Win
- Before / After
- Palette Strip
- Visual Breakdown
- Three-Step Workflow
- Next Up

Body draft:

> Day 5 proof: the same four kitchen props, but now they share one palette. The shapes did not
> change. The set suddenly feels more intentional.
>
> Today's tiny win: I reused the exact same wood, metal, cream, and accent colors across four props,
> and now they look like they belong in one asset pack.
>
> Before: four useful props, but each one felt separate. After: shared wood, shared metal, shared
> cream, and a small warm accent tying them together.
>
> Six colors did the job: cream for ceramic, wood for bodies and handles, dark wood for edges, steel
> for tools, tomato for warmth, and leaf green for a tiny fresh accent.
>
> What changed:
> - Wood: crate, board, and handles now use the same brown.
> - Metal: pot, spoon, and knife share one steel color.
> - Cream: the plate color repeats cleanly.
> - Accents: tomato red and leaf green add life without making the pack messy.
>
> The process in three steps:
> 1. Loaded the Endesga64 course palette in Blockbench.
> 2. Refilled each prop from that same palette.
> 3. Put the four props side by side to check whether they read as one pack.
>
> The mistake/fix:
> The mistake: eyeballing "close enough" browns for each model.
> The fix: type the same HEX value every time.
>
> Next up: painting the first real pixel texture on the crate.
>
> Streak: Day 5.

Images:

- Use `artifacts/palette/lesson5-before-after.png` as the hero proof.
- Add a palette strip image if one exists; otherwise keep the palette text short.

## Filled Example: Day 6 First Pixel Texture

Title:

> Day 6: The crate gets its first pixel texture

Blocks to use:

- Hero Proof
- Tiny Win
- Before / After
- Visual Breakdown
- Mistake / Fix
- Thumbnail Test
- Next Up

Body draft:

> Day 6 proof: same crate, same shape, same palette. The difference is all painted pixels: plank
> seams, darker corners, and a few tiny wood marks.
>
> Today's tiny win: I kept the old crate model and upgraded only the texture, so it reads more like
> wood without adding new cubes.
>
> What changed:
> - Plank seams: three dark lines make the front face feel built from boards.
> - Corners: short darker marks give the box a little edge shadow.
> - Highlights: a couple of light pixels keep it stylized instead of muddy.
>
> The process in three steps:
> 1. Reopened the Day 1 crate.
> 2. Painted with a size-1 brush using the shared palette.
> 3. Zoomed out to check if the texture still reads at thumbnail size.
>
> The mistake/fix:
> The mistake: adding too many marks until the crate looked noisy.
> The fix: remove detail until the big plank read is clear again.
>
> Thumbnail test: if the crate is small and I can still see wood boards, the texture is doing its job.
>
> Next up: building a chalkboard menu stand and learning first UV mapping.
>
> Streak: Day 6.

Images:

- Hero proof: new crate render.
- Before/after: Day 1 flat crate beside Day 6 textured crate.
- Optional close-up: cropped front face showing plank seams.
