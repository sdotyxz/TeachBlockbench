MTools + UV Unwrapping

Use MTools to reshape and generate meshes. For UVs, remember one sentence: Cubic Projection unwraps around six directions; Project from View flattens what the camera sees.

The mental model

UV unwrapping flattens selected 3D faces into 2D coordinates on the texture. It changes neither the geometry nor the painted pixels. After unwrapping, you still arrange the UV islands, snap them to pixels, create or update the texture, and paint it.

If the menu is missing

MTools needs a format that supports meshes, a selected Mesh rather than a Cube, and Edit Mode. A Generic Model is the safest starting point. Object selection shows MTools Operators; any non-object selection mode shows MTools.

The UV choice

MethodWhat it doesBest forAvoid when
MTools: Cubic Projection Groups selected faces by their nearest X, Y, or Z direction, then projects them from the six sides of an imaginary box. Crates, counters, shelves, appliances, and other mostly boxy props. The mesh is rounded, strongly diagonal, or needs carefully connected islands.
Blockbench: UV Project from View Flattens selected faces using the current camera view. Signs, labels, logos, front panels, and one camera-facing surface. You selected back faces too, or the camera sees the surface at a steep angle.
Seam Tool + texture template Marks where connected mesh faces must split or stay joined when Blockbench generates a texture template. Bottles, jars, cups, pipes, and continuous side bands. You only need to place one flat label; Project from View is faster.
Manual UV editing Moves, rotates, scales, connects, and snaps the resulting UV islands. The final cleanup after every automatic projection. Never skip it merely because the first projection looks acceptable.

Recipe: MTools Cubic Projection

  1. Duplicate the mesh or save first

    The command replaces UV coordinates on the selected faces. Geometry stays unchanged, and Undo should work, but a duplicate makes comparison easy.

  2. Use a square texture while learning

    MTools scales the projection from the shorter project texture dimension. A square 32 × 32 or 64 × 64 texture makes the result easiest to predict.

  3. Select only the faces to unwrap

    Enter Edit Mode, select the mesh, choose Face selection, then select the intended faces. Unselected faces keep their existing UVs.

  4. Open MTools → UV Mapping → Cubic Projection

    On mobile, long-press the selected mesh in the Outliner to open its context menu.

  5. Start with Split enabled

    Split places the six directional projections into separate regions. This reduces overlap and is the safer choice when each side needs unique painting.

  6. Add a small Margin

    Start around 2–5%. Margin shrinks each projected group slightly to leave breathing room. Inspect the result at the actual texture resolution; it is percentage-based, not an exact pixel-padding control.

  7. Clean the UV panel

    Move islands apart, rotate sideways pieces, match their texel scale, and use Snap UV to Pixels for crisp pixel art.

  8. Test with a checker or numbered marks

    Paint a few temporary pixels across edges. Distortion or discontinuity becomes obvious before you spend time on the final texture.

Cubic Projection settings

SettingResultChoose it when
Split: On Directional groups occupy separate parts of a roughly 3 × 2 layout. You want to paint the top, sides, front, and back differently.
Split: Off Different directional groups project into the same texture area and may overlap. You intentionally want several sides to reuse the same pixels.
Margin: 0% Uses the full allocated projection region. You need maximum area and will manually add spacing.
Margin: 2–5% Slightly shrinks each group. You want a safer starting gap before painting.
Large margin Shrinks the UV coverage and makes texture pixels appear larger on the model. Rarely useful; watch for wasted texture space and uneven detail scale.
Important limitation

Cubic Projection is a directional projection, not an intelligent island packer. It does not detect the most hidden seams, guarantee even padding, fill every empty pixel, or preserve a carefully hand-arranged layout. Treat its output as a fast first draft.

Recipe: Project from View

  1. Select only the visible-facing faces

    Back and front faces can overlap when projected from the same camera.

  2. Align the camera square to the surface

    Use a front, side, or top orthographic view when possible. Perspective or a steep camera angle creates a skewed UV shape.

  3. Run the built-in command

    Use UV → UV Project from View. On mobile, tap Search and enter UV Project from View.

  4. Place and snap the result

    Move the projection onto the intended label or sign patch, scale it to the correct proportion, and snap its corners to pixels.
Deprecated MTools command

MTools still lists Project From View, but version 2.1.0 marks it deprecated. Prefer Blockbench's built-in UV Project from View; it is the maintained path.

30-second self-check

Which projection should you try first for a boxy café counter?
MTools Cubic Projection, normally with Split enabled.
Which projection should you use for one readable front label?
Blockbench UV Project from View, with the camera square to the surface.
Why would you deliberately turn Split off?
To overlap several directions so they intentionally reuse the same texture pixels.
Does an automatic projection finish the UV work?
No. Arrange, scale, rotate, pad, test, and pixel-snap the resulting islands.
