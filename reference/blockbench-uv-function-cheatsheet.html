Reference - Blockbench UV mapping

Blockbench UV Function Cheatsheet

UV mapping decides which part of a texture image appears on each model face. Painting changes the pixels. UV functions change how those pixels are placed, scaled, rotated, mirrored, or generated on the model.

The one mental model

ThingWhat it meansWhat changes on the model
Texture size The resolution of the image, such as 16 x 16 or 32 x 32. How many pixels are available to paint.
UV position Where a face's UV rectangle sits on the texture image. Which patch of the image appears on the face.
UV size The width and height of that UV rectangle on the image. How zoomed-in or zoomed-out the texture looks on the face.
UV rotation The direction the UV rectangle points. Whether the texture appears upright, sideways, or upside down.
Answer to "what does UV size affect?"

A larger UV size samples more of the texture image across the same model face, so details look smaller and less stretched. A smaller UV size samples fewer pixels, so those pixels stretch across the face and look bigger, chunkier, or blurrier. UV size does not resize the cube and does not change the texture canvas resolution.

Beginner-safe values

Setting or habitStart hereChange it when
Texture size 16 x 16 or 32 x 32 for small props. The prop needs readable labels, icons, or several unique faces.
UV size Match the face's rough shape. Long face = long UV rectangle. The texture is stretched, squashed, too large, or too tiny.
Painting Grid On. Leave it on while learning pixel textures.
Snap UV to Pixels Use after moving UV corners. Edges look half-pixel blurry or tiny pixel details smear.
Face UV vs Box UV Use Face UV when placing one specific patch on one face. Use Box UV when you want a cube's sides laid out as one box-style wrap.

Core UV functions

FunctionWhat it doesUse it when
Auto UV Sets the selected face's UV size to match the real face size. A face is stretched and you want a sensible default.
Relative Auto UV Sets the selected face's UV position and size from the actual face. You want placement to roughly follow the model's position, not just its size.
Reset Face Returns the selected face's UV mapping to a default state. You made a face confusing and want to start that face over.
Maximize UV Maps the selected face to the full texture image. You want one face, such as a sign, to use the entire texture.
Move UV to Cursor Moves selected UV faces to the mouse cursor position. You need to quickly reposition an island to a new patch.
Snap UV to Pixels Snaps selected UV vertices onto the pixel grid. A pixel-art texture looks soft, smeared, or misaligned.
Rotate UV Left / Right Rotates selected UV faces by 90 degrees. The grain, label, or icon direction is sideways.
Turn Mapping Turns the UV mapping around 90 degrees. A cube face is using the right texture area but the orientation is wrong.
Mirror X / Mirror Y Flips the selected UV mapping horizontally or vertically. Text, arrows, or asymmetric details appear backward.
UV Project from View Generates UVs for selected faces from the current 3D camera view. You want a flat projection for a sign, label, front face, or decal-like area.
Connect UV Faces Connects selected UV faces to the last selected UV face. Adjacent model faces should share a continuous texture strip.
Merge UV Vertices Snaps selected UV vertices to the last selected UV vertices. Corners should line up exactly on the texture grid.
Display UV Faces Chooses which UV faces are visible in the UV editor. The UV editor feels crowded and you only want selected or relevant faces.
UV Overlay Shows the UV map as an overlay in Paint Mode. You want to paint while seeing where face boundaries are.
Export UV Mapping Exports the UV layout as a reference image. You want to paint externally or keep a layout guide.

Face UV vs Box UV

ModeBest forWatch out for
Face UV Manually placing one patch on one face, such as the MENU text in Lesson 7. Each face can drift out of alignment if you move them independently.
Box UV Cube-style wrapping where the sides should behave as one unfolded box. Less direct control for a single face-specific label or decal.

Troubleshooting

SymptomLikely causeTry this
Texture looks stretched. The UV size does not match the face proportions. Use Auto UV, then resize the UV rectangle manually if needed.
Texture detail is too large. The UV size is too small. Make the UV rectangle larger so the face samples more pixels.
Texture detail is too tiny. The UV size is too large. Make the UV rectangle smaller so the face samples less of the image.
Text is cut off. The UV position or size misses part of the painted patch. Move the UV rectangle over the patch, then resize until the whole word is inside.
Text is backward or upside down. The UV mapping is mirrored or rotated. Use Mirror X, Mirror Y, Rotate UV, or Turn Mapping.
Pixel art looks soft. UV corners are off the pixel grid, or the texture is being sampled between pixels. Use Snap UV to Pixels and keep the texture small and crisp.
Wrong face changes. The wrong face tab or face selection is active. Orbit the model, select the visible face directly, then edit only that selected face.
