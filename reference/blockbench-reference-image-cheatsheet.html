Reference - building from images without copying blindly

Blockbench Reference Image Cheatsheet

A reference image is a guide in the viewport. It helps you match silhouette, proportion, and details while you model. It is not the texture, and it is not the final screenshot.

Do not mix these up

ThingWhat it isUse it for
Reference image A guide image placed behind, inside, or over the viewport. Checking shape, scale, angle, silhouette, and important features.
Texture image The image mapped onto the model's faces. Color, pixel marks, labels, material detail, and UV mapping.
Render / screenshot The exported view of your finished model. Posting progress, comparing before/after, and showing the asset pack.
Beginner rule

Use one main reference image for the silhouette, then build a simplified prop from it. Do not chase every tiny detail. For this course, the model should read as a clean low-poly kitchen asset, not a perfect copy of the photo.

Where the controls are

ControlPath or actionWhat it does
Edit Reference Images View -> Edit Reference Images Turns on reference image mode so you can add, select, move, resize, and edit references.
Add Reference Image Reference image toolbar, or drag an image into Blockbench. Adds an image as a viewport guide.
Reference Image from Clipboard Reference image toolbar. Adds a copied image without saving it as a separate file first.
Reference Image List Reference image toolbar. Shows project, global, and built-in references with visibility, mode, refresh, delete, and properties options.
Toggle All Reference Images Reference image toolbar. Hides or shows all active reference images at once.
Select Reference Image on Double Click File -> Preferences -> Settings -> Controls Lets a double-click in the viewport select a reference image.

Reference image settings

SettingPlain meaningBeginner advice
Layer Where the image sits: background, viewport, or floating UI layer. Use background or viewport for modeling. Use float only when you want a loose on-screen note.
Scope Whether the image belongs to this project or to the app globally. Use project for lesson-specific photos. Use global for reusable guides like a style board.
Position The image's 2D offset. Center the subject near the model before resizing.
Size The displayed width and height of the reference image. Match the model's intended height or width first; exact scale can come later.
Rotation The image's angle in the viewport. Keep it at 0 unless the reference photo itself is tilted.
Opacity How visible the reference image is. Start around 40-60% so you can see both the model and the guide.
Visibility Whether this reference is shown. Turn it off when checking your model's silhouette without help.
Blueprint Attaches the image to the selected viewport direction. Use for front/side/top orthographic guides when you need clean alignment.
Clear Mode Makes the reference easier to see against the viewport/model. Use it when the image and model visually blend together.
Enabled Modes Controls which Blockbench modes show the reference. Show references in Edit Mode; hide them in Paint Mode if they distract from texture work.

Recommended workflow

  1. Choose a useful image

    Pick a clear front, side, or 3/4 view. Avoid busy photos where the object is hidden, cropped, heavily blurred, or lit so dark that the silhouette is hard to read.

  2. Add it to the project

    Use View -> Edit Reference Images, then add the image. Choose project scope so the guide travels with the lesson file.

  3. Set opacity before modeling

    Lower opacity until the guide stops fighting the model. If the image dominates the viewport, you will copy details instead of designing the asset.

  4. Match the big box first

    Build the largest simple volume that matches the reference: board, pot body, crate, bottle, or basket. Do not start with handles, pegs, labels, or tiny marks.

  5. Check silhouette from a distance

    Hide the reference, zoom out, and ask whether the model still reads. If it only works with the photo visible, simplify the shape.

  6. Use the reference for decisions, not decoration

    Pull out 3-5 important features: chunky rim, angled legs, handle shape, label placement, material split. Leave the rest out.

When to use blueprint mode

SituationUse blueprint?Why
You have clean front and side drawings. Yes. Blueprints help align orthographic views to the model axes.
You have one casual 3/4 photo. No, usually. Use it as a loose visual guide, not a precise measured drawing.
You are making a simple Overcooked-style prop. Only if the shape is hard to judge. Most course props need readable proportions, not technical tracing.
You need exact symmetry or exact scale. Yes. Blueprints make front/side/top alignment less guessy.

Troubleshooting

ProblemLikely causeFix
You cannot select the reference. Reference image mode is off, or double-click selection is disabled. Turn on View -> Edit Reference Images, then use the image list or enable double-click selection.
The image hides the model. Opacity is too high or the layer is too intrusive. Lower opacity, change layer, or toggle visibility while checking the model.
The model looks good only from one angle. You copied a flat photo instead of building a 3D form. Orbit often and use the reference for proportions, not as a 2D tracing sheet.
The reference does not save with the model. It may be global instead of project-scoped. Open properties and set scope to project for lesson-specific references.
The reference is distracting in Paint Mode. It is enabled in too many modes. Use Enabled Modes and keep it mainly in Edit Mode.
The image is tilted or mismatched. The source photo has perspective, rotation, or cropping. Use it loosely, rotate/resize it, or find a cleaner front/side image.
Common mistake

Do not treat a reference image as permission to over-detail. If the reference has 30 tiny screws, stains, labels, and scratches, choose the few details that make the object recognizable at thumbnail size.
