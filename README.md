# TeachBlockbench

A self-paced course for learning **Blockbench** to build a sellable, **Overcooked-style
low-poly asset pack** — shipped to itch.io with a daily public post (itch.io + RedNote/小红书).
~1 hour a day, total beginner → first pack in 1–2 months.

Run with the `/teach` skill. Each session ends with something postable.

## What's here

| Path | What it is |
|------|------------|
| `MISSION.md` | Why this course exists + what success looks like. The compass for every lesson. |
| `RESOURCES.md` | Curated, high-trust sources (official docs, tutorials, communities, selling guides). |
| `NOTES.md` | Teaching preferences + the rough week-by-week roadmap. |
| `lessons/` | The daily lessons — self-contained HTML. **Start with `lessons/0001-…`.** |
| `reference/` | Cheat sheets to keep open while working (interface, navigation, glossary). |
| `assets/` | Shared stylesheet reused by every lesson. |
| `learning-records/` | What's been learned — drives what gets taught next. |

## Start here
Open `index.html` in a browser. It links the current lesson, all existing lessons, and the
reference sheets.

## GitHub Pages
This repo is ready to publish as a static GitHub Pages site from the repo root.

1. Push the repo to GitHub.
2. Open the repository on GitHub.
3. Go to **Settings → Pages**.
4. Set **Source** to **Deploy from a branch**.
5. Choose your main branch and the root folder `/`.

GitHub Pages will use `index.html` as the homepage. The `.nojekyll` file tells GitHub to serve the
static files as-is.
