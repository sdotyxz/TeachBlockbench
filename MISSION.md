# Mission: Overcooked-style game assets in Blockbench

## Why
To learn 3D modeling from scratch and ship a **sellable, Overcooked-style asset pack** on
itch.io — cute chefs, kitchen props, and modular level pieces. The deeper goal is to convert
one focused hour a day into a *public, compounding* body of work — posted daily to itch.io and
RedNote (小红书) — that proves the skill and can eventually earn.

## Success looks like
- Can start from a blank **Generic Model** and model, color, and pose a clean low-poly prop in
  Blockbench, unassisted.
- Has published one cohesive **Overcooked-style asset pack** on itch.io: several props + at
  least one character, a real product page, a clear license, and game-ready export files
  (`.gltf` / `.glb` / `.obj`).
- Holds a **daily public posting streak** across itch.io + RedNote — one postable artifact a day.
- Can export an asset in a format a game engine *could* import (even though we never build the game).

## Constraints
- **~1 hour per day.** Every lesson must fit one sitting and end with something postable.
- Target: a shippable first pack within **1–2 months**.
- **Complete beginner** — no prior 3D modeling, no prior Blockbench.
- Strong preference for the **fastest path to a tangible, shareable win** each session
  (momentum over completeness).
- **Pure Blockbench** — no game engine (Godot/Unity) in scope.

## Out of scope (for now)
- Building a playable game or any engine/gameplay code.
- Rigging and complex character animation (basic posing only, until pack basics are solid).
- High-poly sculpting workflows (Blender sculpt, ZBrush).
- Hand-painted high-res / PBR texturing — we stay low-poly with simple, pixel-style textures.
