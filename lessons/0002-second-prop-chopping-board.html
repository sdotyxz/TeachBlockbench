Warm up — 30-second recall

Answer each out loud from memory before you open it. The retrieval is what makes Lesson 1 stick — don't skip it.

Which project type do you start, every time?
Generic Model. Your default for all game art.
The three camera moves — and the one that always works?
Rotate (left-drag), Pan (right-drag), Zoom (scroll — works everywhere).
Why was your crate white until you made a texture?
Color lives in a texture painted onto faces. No texture → plain white.

The one new idea

A real prop is just several cubes positioned next to each other. That's the whole secret to everything you'll build from here — a chef, a stove, a counter: all of it is boxes placed together. The only new tool you need is the Outliner — the list of every cube in your model. As soon as you have more than one cube, name them, or you'll get lost in Cube, Cube2, Cube3. The Outliner is your parts list. [Blockbench Wiki — panels]

▭ + ▬ + ▪ = 🔪🪵
Three cubes — a flat board, a thin blade, a small handle — read as one little scene.

Do it with me

Same loop as Lesson 1 (add cube → shape → color) — just repeated three times, with names. Keep the Cheat Sheet open.

  1. New Generic Model → name it chopping-board

    You know this one: File → New → Generic Model. Empty grid, fresh start.

  2. Build the board: one flat, wide slab

    Add a cube. In the Element panel, type a flat-slab Size — roughly Width 14 · Height 1.5 · Depth 9 (eyeball it; numbers don't need to be exact). Sit it on the grid floor. A cutting board is just a squashed crate.

  3. Name it in the Outliner

    Find the new cube in the Outliner panel, double-click its name, and call it board. This is the new habit: every cube gets a name the moment it exists.

  4. Add the knife handle (cube #2)

    Add another cube. Make it a small bar — about Width 4 · Height 1 · Depth 1.2. Switch to the Move tool (Space flips Move ⇄ Resize) and drag it so it rests on top of the board, near one short edge. Rename it handle.

  5. Add the blade (cube #3)

    Add a third cube. Make it thin and flat — about Width 6 · Height 0.4 · Depth 2 — and Move it so it lines up end-to-end with the handle, lying flat on the board. Now the two cubes read as one knife. Rename it blade. You should see board · handle · blade in the Outliner — that's a tidy model.

  6. Color all three (your Lesson-1 loop)

    Create a texture (16 × 16), switch to Paint mode, and click faces to fill — exactly like the crate, just more faces:
    • board → a warm wood brown. Tip: match it to your crate's brown so the two props look like a set.
    • blade → light steel grey.
    • handle → a darker brown or black.

  7. Frame a 3/4 shot & capture it

    Back to normal view, orbit to that slightly-looking-down corner angle, and screenshot. Bonus: put the crate and the board in one shot later for a "set" photo.

  8. Post your Day-2 devlog

    Same project page as yesterday. Screenshot + a caption from below. Streak: Day 2.

Quick self-check

From memory first — this is today's new material, so the recall matters most here.

Where do you rename a cube, and why bother?
In the Outliner. Named parts keep a multi-cube model from becoming an unreadable pile as it grows.
What makes a thin cube read as a knife blade?
A very small Height — a flat, wide, shallow box. Shape comes from proportions, not detail.
How do you place one cube against another precisely?
The Move tool (Space to switch), or type exact Position numbers in the Element panel.
What turns two separate props into a sellable "set"?
A shared look — same palette and blocky style — so they belong together.
