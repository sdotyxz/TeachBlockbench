The one new idea — a limited, shared palette

A palette is the fixed set of colors a whole project is allowed to use. The course palette is Endesga64, a 64-color pixel-art palette. For today's beginner pass, you will use the six starter role colors below. When every wooden thing is the same brown and every metal thing the same grey, separate objects start to read as one family. A buyer scanning itch.io for a "pack" is really looking for that cohesion — it's what separates a product from a pile of models.

Blockbench has this built in. The Color panel has a Color Bar (your current color + its HEX value + recent history) and three tabs: Picker, Palette, and Both. The Palette tab lets you add, save, import, export, and load palettes, and there's an “Add To Palette” action plus a “Pick Screen Color” sampler. [Blockbench Wiki — Color panel]

Your palette is already chosen for you. Keep the Course Palette reference open — it is the single source of truth for color from here on.

Why fill, not eyeball

The Paint Bucket tool fills "faces, cubes, or connected areas with a single color," depending on its Fill Mode. Setting Fill Mode to Cube recolors a whole part in one click — fast, and exact every time, so two props end up with the truly identical brown. [Wiki — paint tools]

Do it with me

Open both references: the Course Palette and the Interface Cheat Sheet.

  1. Open one prop and find the Color panel

    File → Open your crate.bbmodel. Find the Color panel (the one with a color square and a HEX value). Click the Palette tab.

  2. ⭐ Load Endesga64, then pin today's starter colors

    In the Palette tab, look for Load Palette → Endesga 64. If it is available, load it. Then find these six course starter colors: F9E6CF, BF6F4A, 5D2C28, 92A1B9, EA323C, 5AC54F. If your Blockbench build does not show Endesga 64, type those six HEX values into the Color Bar and use Add To Palette for today.

  3. Save the palette so it survives

    In the Palette tab, use Export to save the palette to a file (e.g. endesga64-course-palette). In each later prop you can Import it back — so every model in the pack literally shares the same color source.

  4. Recolor the crate → Wood

    Switch to Paint mode. Pick the Paint Bucket tool and set its Fill Mode to Cube. Click the Wood swatch (BF6F4A), then click the crate. The whole part fills with the shared brown. Optional: a couple of Dark Wood (5D2C28) clicks on edge slats for depth.

  5. Recolor the other three (use the reference table)

    Open each remaining prop and fill from the same palette:
    • Chopping board → Wood body, Steel knife blade, Dark Wood handle.
    • Cooking pot → Steel body, Tomato or Dark Wood handles.
    • Plate & spoon → Cream plate, Steel spoon, Leaf garnish.
    Re-save (Ctrl/Cmd S) each .bbmodel after recoloring.

  6. Frame the before / after set shot

    Put all four props in one scene and take a clean shot. If you still have an old screenshot from before recoloring, pair them side by side — the “they match now!” jump is the whole post.

  7. Post your Day-5 devlog

    Open your itch.io project, create a new devlog post, upload the before/after set shot, paste the English caption below, and publish. The post does not need to be perfect: one clear image, one short paragraph, one public proof-of-work entry. Streak: Day 5.
Shortcut for matching to a real image

Want colors pulled straight from a reference image? Use Pick Screen Color to sample a pixel directly, then choose the nearest Endesga64 swatch. Great for later, when you adapt a reference without breaking the course palette.

Quick self-check

Say the answer first, then open the card.

What does a limited, shared palette do for a pack?
It creates cohesion — repeated colors make separate props read as one designed set, which is what buyers want.
How do you save a color into Blockbench's palette?
Set the color in the Color Bar (type the HEX), then use Add To Palette. Export the palette to reuse it across files.
Which tool + setting recolors a whole part in one click?
The Paint Bucket with Fill Mode → Cube. One click fills the entire part with the chosen color.
Why type exact HEX codes instead of eyeballing a brown?
So two different props get the identical color. "Close enough" browns break the illusion that the props belong together.
