13First Timeline animation - a bouncing tomato loop

You have a growing set of static kitchen props. Today you make the first moving asset-pack proof: one tomato group that bounces with three position keyframes in Blockbench's Timeline.

~45-60 min Builds on Lessons 4, 9, and 11 Generic Model supports Animate mode Ends with a GIF post
Today's win

One 1-second looping GIF: a low-poly tomato bouncing on a tiny counter tile, made from three keyframes: down, up, down.

Lesson target: the first and last keyframes match, so the loop can repeat without a visible jump.

Warm up - 30-second recall

Answer first, then open the cards.

What is the difference between rotating the camera and rotating an element?
Camera rotation only changes your view. Element rotation changes the model itself.
Why did the tomato trio use Cluster Selection?
Cluster selected one connected tomato chunk without selecting every face by hand.
Why do we name parts in the Outliner?
Names help you select the exact part or group you want to edit later.

The one new idea - keyframes save poses over time

An animation is not a different model. It is the same model with saved poses at different times. Each saved pose is a keyframe. Blockbench plays between those keyframes in the Timeline.

For the first animation, keep the idea tiny: save the tomato down at 0.0s, up at 0.5s, and down again at 1.0s. The first and last keyframes match, so the loop can restart cleanly.

Beginner boundary

Do not animate every cube separately today. Put the moving tomato pieces in one group, then animate the group. One moving group is easier to understand than ten moving parts.

Do it with me

  1. New Generic Model - name it bouncing-tomato-loop

    Save early. This is a motion study, not a full new prop challenge.

  2. Build a tiny counter tile

    Add one flat cube as a simple base. Suggested size: X 5, Y 0.25, Z 4. Color it with the shared wood colors or reuse the Lesson-9 prep counter idea.

  3. Build one simple tomato above the tile

    Use a low-poly mesh sphere if you are comfortable from Lesson 11, or use a chunky red cube if you want the fastest path. Add a tiny green stem. Keep it readable at thumbnail size.

  4. Group the moving tomato parts

    Put the tomato body and stem into one Outliner group named tomato_bounce_group. Leave the counter tile outside the group so the tile stays still.

  5. Switch to Animate mode

    Use the mode switcher to enter Animate. If the Timeline is hidden, open View -> Panels -> Timeline. If the Animations panel is hidden, open View -> Panels -> Animations.

  6. Add one animation

    In the Animations panel, add a new animation. Name it tomato_bounce. Set the length to 1.0 second if the length field appears.

  7. Select the moving group

    Select tomato_bounce_group. In the Timeline, you want to key the group, not the counter tile and not one small stem cube.

  8. Keyframe 1: down at 0.0s

    Move the Timeline playhead to 0.0s. Add a Position keyframe for the tomato group. This saves the starting pose.

  9. Keyframe 2: up at 0.5s

    Move the playhead to 0.5s. Move the tomato group up a small amount, about Y +0.7. Add another Position keyframe.

  10. Keyframe 3: down at 1.0s

    Move the playhead to 1.0s. Put the tomato back at the same position as 0.0s. Add the final Position keyframe. Matching the first pose prevents a pop.

  11. Turn on Looped Playback and preview

    Play the Timeline. The tomato should bounce up and return down, then repeat. If it jumps at the end, the 1.0s keyframe does not match the 0.0s keyframe.

  12. Add one optional squash illusion

    Optional: at the down keyframes, make the tomato slightly wider and lower. Only do this if the simple bounce is already working. Save before experimenting.

  13. Export or capture a GIF

    Capture a short looping GIF or screen recording of the Timeline playback. Keep the camera fixed so viewers notice the tomato movement, not the camera movement.

  14. Post your Day-13 update

    Publish the GIF and a short note. Streak: Day 13.
Animation habit

For every loop, check this first: does the last keyframe match the first? If yes, the loop can repeat cleanly.

Quick self-check

Say the answer first, then open the card.

What is a keyframe?
A saved pose or value at a specific time in the animation.
Why animate the group instead of every cube?
One grouped animator keeps the first animation simple and makes the whole tomato move together.
Why should the first and last keyframes match?
A matching start and end lets the loop restart without a visible jump.
If nothing moves during playback, what do you check first?
Check that you are in Animate mode, the right group is selected, and Position keyframes exist on that group.
